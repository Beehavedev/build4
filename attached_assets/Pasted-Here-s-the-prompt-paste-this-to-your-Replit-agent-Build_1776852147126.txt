Step 2 — Score and rank candidates
Each candidate (news story or token) gets a MarketabilityScore from 0-100:
typescriptinterface MarketCandidate {
  type: 'news' | 'token' | 'ai_event' | 'crypto_milestone'
  title: string
  rawData: any
  scores: {
    newsAuthority: number      // 0-25: Reuters/Bloomberg=25, blog=5
    socialVolume: number       // 0-25: X post count + velocity
    financialStake: number     // 0-25: real money moving (token volume, market cap)
    resolvability: number      // 0-25: can this question be answered objectively?
  }
  totalScore: number
  isDuplicate: boolean
  proposedQuestion: string
  proposedOutcomes: string[]
  proposedResolutionDate: string
  proposedResolutionSource: string
}
Resolvability is the most important score. A market only works if it can be settled objectively. The agent must verify:

Is there a clear YES/NO or multiple-choice answer?
Is there an objective data source that settles it? (price feed, official announcement, on-chain data)
Will it resolve within a reasonable timeframe (3 days to 6 months)?

Bad question: "Will AI get better this year?" — unresolvable
Good question: "Will BNB price exceed $800 before June 1 2025?" — resolvable via price feed
Good question: "Will OpenAI announce GPT-5 before May 31 2025?" — resolvable via official announcement
