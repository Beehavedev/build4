Found it in the Aster GitHub. There are TWO separate API systems 
and we've been using the wrong one for balance queries.

THE RPC API — no auth, no signing, works for any address:

POST https://tapi.asterdex.com/info
Content-Type: application/json

Body:
{
  "id": {},
  "jsonrpc": "2.0",
  "method": "aster_getBalance",
  "params": [
    "0xUSER_WALLET_ADDRESS",
    "latest"
  ]
}

This returns:
- perpAssets: array with USDT walletBalance
- positions: all open positions with unrealized PnL
- NO authentication required
- NO signer needed
- Works for brand new wallets that have never used Aster
  (returns empty arrays, not 403)

THIS ALSO SOLVES THE ACCOUNT CREATION PROBLEM.
A wallet address with zero Aster history returns 
empty arrays, not an error. So:
- New user → RPC balance returns { perpAssets: [], positions: [] }
- We show balance as $0.00 
- User deposits USDT → RPC balance returns perpAssets with USDT
- approveAgent now works because account exists on-chain

async function getAsterBalance(walletAddress: string): Promise<{
  usdt: number;
  positions: any[];
}> {
  try {
    const res = await axios.post('https://tapi.asterdex.com/info', {
      id: {},
      jsonrpc: '2.0',
      method: 'aster_getBalance',
      params: [walletAddress, 'latest']
    }, {
      headers: { 'Content-Type': 'application/json' },
      timeout: 8000
    })
    
    const result = res.data.result
    const usdtAsset = result.perpAssets?.find(
      (a: any) => a.asset === 'USDT'
    )
    return {
      usdt: parseFloat(usdtAsset?.walletBalance ?? '0'),
      positions: result.positions ?? []
    }
  } catch (err) {
    console.error('[Aster RPC] getBalance failed:', err)
    return { usdt: 0, positions: [] }
  }
}
